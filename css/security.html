Article 2 - Security on the Web/Internet

Common Risks & Safe Practices | Last updated: October 12, 2025

Risks, Technologies, and Best Practices

The weakest link in cybersecurity is the human being… Think before you click.

Introduction

The Internet can be defined as a global system of interconnected computer networks that communicate using Internet Protocols (IP).

Web security involves a combination of technologies, protocols, and practices designed to protect websites, web applications, servers, and online data from internet-based cyberattacks like malware, phishing, and data breaches. These threats may compromise data confidentiality, integrity, or availability [2.1].

As reliance on digital services grows, web security has become a critical concern for individuals, businesses, and governments worldwide.

Common Risks on the Web

One of the most significant risks is malware, a general term for malicious software such as viruses, worms, and ransomware. This malicious software infiltrates systems to cause damage or steal information [2.2]. Attackers often use techniques like phishing, where deceptive emails or websites trick users into revealing sensitive information such as passwords or financial details [2.3].

Another type of threat is the Distributed Denial of Service (DDoS) attack, where attackers overwhelm a website or server with excessive traffic, making it unavailable to legitimate users [2.2]. Similarly, web application vulnerabilities such as SQL injection and cross-site scripting (XSS) allow attackers to manipulate databases or inject malicious code into web pages [2.4].

Data privacy is at risk through man-in-the-middle (MITM) attacks, where an attacker intercepts communication between a user and a website, capturing login credentials or sensitive data [2.3]. As the Internet of Things (IoT) expands, poorly secured devices further increase the attack surface [2.5].

Safe Practices for Users and Organizations

To counter these risks, users and organizations must adopt security practices.

Individuals must always use strong and unique passwords; it is an essential measure. A strong password typically contains a mix of uppercase and lowercase letters, numbers, and special characters, and should never be reused across services [2.2]. Multi-Factor Authentication (MFA), which requires additional verification beyond a password, provides another layer of defense.

Organizations should prioritize Secure Socket Layer / Transport Layer Security (SSL/TLS) encryption to protect transmitted data between users and servers [2.4]. SSL/TLS are cryptographic protocols that provide security for internet communications by encrypting data between a client and a server, ensuring privacy, authenticity, and data integrity.

Regular software updates and patches are critical in closing vulnerabilities that attackers might exploit. Firewalls and intrusion detection systems help monitor and block suspicious activities on networks.

Education and awareness also play a key role. Employees and students trained in recognizing phishing attempts and suspicious links are less likely to fall victim to attacks [2.5]. Adopting the principle of least privilege, limiting user access rights to only what is necessary, reduces the potential impact of compromised accounts.

Emerging Trends in Web Security

Zero Trust Architecture (ZTA) has emerged as a leading security model. Unlike traditional defenses, ZTA assumes that no user or device is inherently trustworthy and continuously verifies identity and access privileges [2.6].

Additionally, the growing use of artificial intelligence (AI) and machine learning allows security systems to detect anomalies and respond to threats in real time [2.6].

The rise of cloud computing also requires enhanced web security strategies. Cloud providers and clients must collaborate to ensure secure configurations, compliance with regulations such as GDPR, and robust data backup mechanisms [2.7].

Conclusion

Web security is a continuously evolving challenge. As new technologies and services expand the digital ecosystem, attackers also develop more advanced techniques. Protecting users and data requires a layered approach that combines technical safeguards, user education, and proactive policy development. Ultimately, maintaining a secure web environment is not a one-time effort but an ongoing process of vigilance and adaptation.
